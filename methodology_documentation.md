# Governance Genome Index (GGI) — Methodology Documentation

This document explains how India Governance Watch scores individual policies and ministries across six established policy-science frameworks, and how those scores are combined into a single composite index — the **Governance Genome Index (GGI)**.

It covers: the theoretical basis of each framework, how raw scores are produced, the exact formulas used to normalize each framework into a 0–100 sub-index, how those sub-indices combine into the GGI, how confidence is tracked, and how ministry-level averages are computed.

**Note on current UI surfacing**: the GGI is fully implemented and computed (`app/services/genome_index.py:compute_ggi`), and available via the API — `ggi` on `GET /api/items/{id}/governance`, and the ministry-wide average via `GET /api/governance/ministries/{id}/genome-index`. It is not currently rendered in the dashboard UI (the Wickedness Index and Governance Genome panels were removed from both the item-detail Governance Intelligence tab and the ministry aggregate view at the user's request). This document remains accurate as the methodology reference for the underlying computation.

---

## 1. The six base frameworks

Every tracked policy item is run through two independent analytical pipelines — **Policy Intelligence** (Lowi's Typology, Civic Engagement, Policy Lifecycle, Implementation Analysis, Lindblom's Incrementalism Scale, and SCTP) and **Governance Intelligence** (Kingdon's Multiple Streams, Punctuated Equilibrium, Policy Entrepreneurs, and the Wickedness Index). The GGI draws on six of these frameworks — the ones that map cleanly onto a single, well-defined analytical dimension:

### 1.1 Lowi's Typology of Policy
Theodore Lowi's classic framework classifies policy by the type of coercion it exercises and the arena of politics it activates: **regulatory** (constrains individual conduct), **distributive** (allocates benefits to specific recipients without direct cost to others), and **redistributive** (shifts resources between broad societal groups, and therefore the highest-stakes, most politically contested arena). Every policy item is scored 0–1 on all three dimensions simultaneously (they aren't mutually exclusive — most policies have some of each character), with an LLM-assigned `dominant_type` for display purposes.

### 1.2 Civic Engagement Dimensions
Drawn from the policy-instruments literature (Schneider & Ingram's tool typology), this framework scores how a policy engages the public across five instrument types: **educate** (information/awareness), **persuade** (behavioral nudging), **coerce** (mandates/penalties), **strengthen** (capacity-building/institutional support), and **incentivize** (financial/material rewards). Each is scored independently 0–1.

### 1.3 Policy Lifecycle Stage
The standard public-policy process model: **Problem Identification & Agenda Setting → Policy Formulation → Legitimation & Adoption → Implementation → Evaluation → Maintenance, Succession & Termination**. Each item is assigned exactly one current stage.

### 1.4 Lindblom's Incrementalism Scale
Charles Lindblom's classic distinction between **"muddling through"** (incremental, adaptive, politically pragmatic policymaking that makes small adjustments to the status quo) and **rational-comprehensive planning** (systematic, analytically-driven policymaking that considers a full range of alternatives from first principles). Scored 0 (pure incrementalism) to 1 (pure rational-comprehensive) on a single continuous scale.

### 1.5 Kingdon's Multiple Streams Framework
John Kingdon's model of agenda-setting: policy change happens when three independent "streams" — the **problem stream** (is there a recognized problem?), the **policy stream** (is there an available, technically feasible solution?), and the **politics stream** (is there political will/opportunity?) — converge, opening a **policy window**. Each stream is scored 0–5, plus a boolean `window_open` flag.

### 1.6 Punctuated Equilibrium Theory
Baumgartner & Jones's model of policy change as long periods of stability ("policy monopolies") punctuated by rapid, disruptive change. Each item is classified into one of four stages: **Policy Monopoly (Stasis) → Image Erosion & Venue Shopping → Positive Feedback Punctuation → New Equilibrium (Post-Punctuation)**.

## 2. How raw scores are produced

All six frameworks' raw scores are generated by a single structured-output call to Google's Gemini model per pipeline (one call for the four Policy Intelligence frameworks that feed the GGI, one for the two Governance Intelligence frameworks), using a fixed JSON response schema so the model cannot omit or malform a field. Prompts are built in `backend/app/services/policy_frameworks.py` (Policy Intelligence) and `backend/app/services/governance_frameworks.py` (Governance Intelligence), grounded with live web-search context (via Serper) about the specific policy before scoring.

Every numeric score is defensively clamped into its valid range server-side after the LLM call returns (`_clamp_result` in `backend/app/services/policy_intelligence.py` and `backend/app/services/policy_governance.py`) — the model is never trusted to self-enforce numeric bounds.

## 3. GGI sub-index formulas

The GGI is built from six sub-indices, each normalized to a common **0–100 scale** so frameworks with different native scales (0–1, 0–5, categorical) can be compared and combined. All math is **deterministic Python** (`backend/app/services/genome_index.py`) — never LLM-generated. This is a deliberate design choice: the composite score must be reproducible and auditable from the underlying raw scores, not subject to LLM variance.

| Sub-Index | Full name | Source | Formula | Rationale |
|---|---|---|---|---|
| **LTI** | Lowi Typology Index | Lowi's Typology | `(0.25 × regulatory + 0.35 × distributive + 0.40 × redistributive) × 100` | Weighted toward redistributive policy, which Lowi's own arena theory treats as entailing the broadest societal negotiation and highest political stakes of the three types. |
| **CEI** | Civic Engagement Index | Civic Engagement Dimensions | `(0.25 × educate + 0.25 × persuade + 0.20 × strengthen + 0.15 × incentivize + 0.15 × (1 − coerce)) × 100` | Weights participatory, capacity-building instruments (educate, persuade, strengthen) above transactional/coercive levers. Coercion is *inverted* (`1 − coerce`) because higher coercion represents lower civic-engagement quality in this index's framing — a policy that mandates rather than persuades scores lower on civic engagement, by design. |
| **PLMI** | Policy Lifecycle Maturity Index | Policy Lifecycle Stage | Ordinal position in the 6-stage cycle, scaled 0–100: Problem Identification=0, Formulation=20, Legitimation=40, Implementation=60, Evaluation=80, Maintenance/Succession/Termination=100 | Reflects how far the policy has progressed through the standard policy process — a proxy for procedural maturity, not policy quality. |
| **LII** | Lindblom Incrementalism Index | Lindblom's Scale | `lindblom_score × 100` | Direct rescale — the underlying score is already 0 (incremental) to 1 (rational-comprehensive). |
| **KMSI** | Kingdon Multiple-Streams Index | Kingdon's Streams | `((problem_score + policy_score + politics_score) / 15) × 100` | Mean stream-alignment strength across the three 0–5-scored streams. `window_open` is reported alongside the score as contextual information rather than folded into the formula with an arbitrary bonus — keeping the arithmetic transparent. |
| **PEI** | Punctuated Equilibrium Index | Punctuated Equilibrium | Ordinal position on the 4-stage stability→disruption cycle, scaled 0–100: Policy Monopoly (Stasis)=10, Image Erosion & Venue Shopping=40, Positive Feedback Punctuation=75, New Equilibrium (Post-Punctuation)=100 | Reflects where the policy domain sits on the stability-to-disruption spectrum. |

### Missing-data handling
The Policy Intelligence and Governance Intelligence pipelines run independently and can complete at different times. If Policy Intelligence hasn't been generated yet for an item, **LTI, CEI, PLMI, and LII default to a neutral 50** rather than being guessed or omitted — the same graceful-degradation convention already used elsewhere in the app for the same situation. KMSI and PEI always have a value once Governance Intelligence exists, since they're sourced from that same pipeline.

## 4. The composite GGI formula

```
GGI = (LTI + CEI + PLMI + LII + KMSI + PEI) / 6
```

An **equal-weighted arithmetic mean** across all six sub-indices. Equal weighting was chosen deliberately as the most defensible default: each of the six frameworks is an independently-grounded, well-established political-science theory, and there is no principled *a priori* basis to weight one framework's contribution above another's. This is documented explicitly here so the weighting can be revisited transparently if a future analytical need calls for it.

### Score interpretation bands

| Range | Label |
|---|---|
| 0–33 | Low |
| 34–66 | Moderate |
| 67–100 | High |

These bands describe the *composite* GGI score's position, not any individual sub-index.

## 5. Confidence propagation

Every framework's LLM output carries its own `confidence` rating (`low` / `medium` / `high`), reflecting the model's self-assessed certainty given the available research context. The GGI's overall confidence is the **worst-case (lowest) confidence** across all contributing frameworks' individual confidence ratings — if any one framework's read is low-confidence, the composite is reported as low-confidence too, rather than averaging away a genuine uncertainty signal.

## 6. Ministry-level aggregation

When viewing a ministry's aggregate governance profile (Ministries tab → select a ministry), the same per-item GGI computation is run against every policy item attributed to that ministry that has Governance Intelligence generated, and the results are **arithmetically averaged** across the ministry's item set:

- Each of the six sub-indices (LTI, CEI, PLMI, LII, KMSI, PEI) is averaged independently across all qualifying items.
- The composite `avg_ggi_score` is the mean of each item's own composite GGI (equivalently, the mean of the six averaged sub-indices, since the composite is itself an equal-weighted mean).
- Kingdon's Streams is additionally reported as raw framework averages (`avg_problem`, `avg_policy`, `avg_politics` on the original 0–5 scale, plus the percentage of the ministry's items with an open policy window) — not just folded into KMSI — so the underlying signal is visible, not just its normalized index form.
- Punctuated Equilibrium is additionally reported as a **distribution** across the four stages (count of items per stage) — an average of an ordinal category label would be misleading, so the ministry view shows the actual spread instead.
- Lowi's Typology, Civic Engagement, and Policy Lifecycle each have their own dedicated ministry-average visuals reusing the app's existing cross-policy aggregate endpoints, shown alongside the GGI breakdown for full transparency into what's driving the composite.

Sample size (`sample_size`) is always reported alongside every ministry aggregate so a low-N average isn't mistaken for a statistically robust signal.

## 7. Data provenance and reproducibility

The GGI is **never stored** in the database and **never generated by an LLM**. It is computed fresh, in pure Python, every time it's requested — directly from the already-persisted `intelligence` and `governance` subdocuments on a policy item (`app/services/genome_index.py:compute_ggi`). This has two consequences worth stating explicitly:

1. **The formula can be revised at any time** without needing to re-run any LLM generation or backfill any stored data — every existing policy item's GGI is recalculated under the new formula the next time it's viewed.
2. **Any governance record generated before the GGI feature existed still produces a fully valid GGI**, as long as its `governance.streams` and `governance.punctuated_equilibrium` fields are present — which they always are, since GGI only reads fields that predate this feature. No migration was required to introduce GGI.

This mirrors the app's general design principle for derived/composite fields (also used for the underlying Governance Genome vector and Wickedness Index overall score): **never trust the LLM to keep a derived number consistent with its own component scores — compute derived numbers in code, from the LLM's raw component outputs.**
